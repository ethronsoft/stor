◆ esft_stor_node_as_stringl()

CSTOR_API const char* esft_stor_node_as_stringl ( esft_stor_node_t *  node,
size_t *  str_len 
)

returns the string representation of the node and sets str_len to the actual length of the string.

The actual length refers to the entire string, which may be exceed the first \0 character encountered. Having this length returned is necessary, as JSON can contain \0 (escaped as , different from \0 required by C), according to RFC 4627. If you know that the returned string will be a normal C string, then just invoke esft_stor_node_as_string

The node must be a value node, such that esft_stor_node_is_string(node *) == true. Calling this function otherwise results in undefined behaviour

TRANSFER OWNERSHIP ON SUCCESS: NO
